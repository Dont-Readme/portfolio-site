## 1. 프로젝트 한 줄 요약

문제 정의부터 화면 기획, 구현 협업, 결과 정리까지 한 흐름으로 보여주기 위한 샘플 프로젝트입니다.

## 2. 배경 / 문제

채용 담당자가 프로젝트 결과물만 보고는 지원자가 어떤 문제를 발견했고 어떤 판단으로 실행했는지 파악하기 어렵다는 문제가 있었습니다.

## 3. 목표

프로젝트를 단순 결과물 목록이 아니라 문제, 역할, 기획 과정, 기능, 성과가 이어지는 사례로 정리하는 것을 목표로 했습니다.

## 4. 내 역할

- 프로젝트 문제 정의
- 사용자 흐름과 화면 구성 정리
- 핵심 기능 우선순위 설정
- 구현 구조와 콘텐츠 구조 연결

## 5. 기획 과정

먼저 채용 담당자가 확인해야 하는 정보를 8개 고정 섹션으로 나누고, 각 섹션에서 어떤 판단 근거를 보여줄지 정리했습니다.

## 7. 핵심 기능

- 프로젝트 메타데이터 기반 카드 목록
- slug 기반 상세 페이지
- MDX 기반 프로젝트 본문 관리
- 섹션별 scroll reveal 애니메이션

## 8. 결과 / 성과

포트폴리오 초기 구조를 빠르게 검증할 수 있고, 실제 프로젝트 자료를 MDX로 교체하기 쉬운 기반을 만들었습니다.
