## 1. 프로젝트 한 줄 요약

사용자 흐름을 기준으로 기능 우선순위를 조정하고 협업 합의 과정을 정리한 샘플 프로젝트입니다.

## 2. 배경 / 문제

초기 요구사항은 많았지만 어떤 사용 흐름이 가장 중요한지 분명하지 않아 화면과 기능 범위가 넓어질 위험이 있었습니다.

## 3. 목표

핵심 사용자 여정을 먼저 정하고, 채용 담당자가 역할과 의사결정 과정을 이해할 수 있도록 프로젝트 서사를 정리했습니다.

## 4. 내 역할

- 사용자 흐름 정의
- 요구사항 분류
- 화면 정책 정리
- 협업 이슈와 결정 사항 문서화

## 5. 기획 과정

기능을 중요도와 구현 난이도 기준으로 나누고, MVP에서 반드시 보여줘야 하는 사용자 행동을 중심으로 화면 구조를 재배치했습니다.

## 7. 핵심 기능

- 사용자 흐름 기반 프로젝트 설명
- 역할과 기간 메타데이터 노출
- 태그 기반 빠른 맥락 제공
- 상세 페이지로 이어지는 카드 UX

## 8. 결과 / 성과

프로젝트를 보는 사람이 결과뿐 아니라 어떤 기준으로 범위를 정했는지 확인할 수 있는 구조를 확보했습니다.
