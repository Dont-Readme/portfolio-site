## 1. 프로젝트 한 줄 요약

프로젝트 상세 콘텐츠를 MDX로 관리하고 slug 기반 라우팅으로 렌더링하는 샘플 프로젝트입니다.

## 2. 배경 / 문제

포트폴리오 프로젝트 설명은 길어질 수 있고 이미지, 리스트, 강조 문구가 섞이기 때문에 일반 데이터 파일만으로 관리하기 불편할 수 있습니다.

## 3. 목표

프로젝트 메타데이터는 TypeScript로, 긴 상세 본문은 MDX로 나누어 관리하는 구조를 검증했습니다.

## 4. 내 역할

- 프로젝트 데이터 모델 정의
- MDX 파일 구조 설계
- 상세 페이지 템플릿 구성
- 빌드 검증 기준 정리

## 5. 기획 과정

프로젝트 상세의 8개 섹션을 고정하고, 각 MDX 파일이 같은 순서로 정보를 제공하도록 템플릿을 구성했습니다.

## 7. 핵심 기능

- `content/projects/*.mdx` 기반 본문 관리
- 정적 slug 생성
- 없는 slug에 대한 not-found 처리
- 공통 ProjectDetailLayout 적용

## 8. 결과 / 성과

실제 프로젝트 자료가 준비되면 MDX 파일과 데이터 메타데이터만 교체해 상세 페이지를 확장할 수 있습니다.
