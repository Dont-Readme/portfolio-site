# AI 학습 코치 알고리즘 데모

## 목적

`/ai-coach-demo2`는 과제 면접에서 추천 로직이 실제 원천 데이터로부터 계산되는 과정을 보여주는 일회성 데모다. 로그인, DB, 학습 완료 후 재정렬, 피드백, RAG, 실제 A/B 테스트는 포함하지 않는다.

화면 결과는 고정된 기준 시점의 합성 학습 데이터로 만든다. 실제 사용자 성과나 출시 후 KPI를 의미하지 않는다.

## 실행

```bash
pnpm install
pnpm dev
```

브라우저에서 `http://localhost:3000/ai-coach-demo2`로 직접 접속한다. Home과 전역 메뉴에는 데모 링크가 없다.

## 환경변수

`.env.example`을 `.env.local`로 복사하고 실제 API를 확인할 때만 아래 값을 채운다.

```env
OPENAI_API_KEY=
OPENAI_MODEL=
```

- 두 변수는 Route Handler에서만 읽으며 클라이언트 번들에 포함하지 않는다.
- 클라이언트는 `fixtureId`만 서버에 전달한다.
- 키 또는 모델이 없으면 유료 API를 호출하지 않고 결정론적인 `정상 AI 응답 가정 시나리오`를 사용한다.
- 실제 API 또는 응답 구조 검증이 시도당 20초 안에 완료되지 않거나 실패하면 한 번 재시도한다. 다시 실패해도 05~07은 같은 `정상 AI 응답 가정 시나리오`로 AI 추천 시연을 이어가고, 실제 호출 실패와 재시도 상태는 08 실행 기록에 별도로 표시한다.
- 시도당 20초와 1회 재시도는 면접 데모에서 실제 API 결과를 확인하기 위한 설정이다. 운영 기획의 추천 반영 목표 시간 2초와는 별도이며, 운영 구현값으로 해석하지 않는다.

## 데모 흐름

1. 합성 사용자 10명 중 한 명을 선택한다.
2. 원천 문제 풀이 이벤트에서 중복, 미채점, 매핑 누락을 제외한다.
3. 최근 5회와 누적 오답률로 개념 취약도와 신뢰도를 계산한다.
4. 상위 취약 개념 3개에서 콘텐츠를 조회한다.
5. 비활성, 수준 차이, 최근 7일 완료 정책을 적용한다.
6. 시스템이 콘텐츠 유형, 예상 학습 시간, 사용자 수준을 정형 데이터로 비교해 `format_fit`, `duration_fit`, `level_fit` 점수를 계산한다.
7. AI는 최대 10개 후보를 한 번에 받아 사용자 학습 목표와 콘텐츠 학습 목표의 의미 관계만 판단하고, 후보별 `goal_relation`, `comment_reason`, `comment`를 반환한다.
8. 서버가 `goal_relation`을 `goal_fit` 점수로 매핑하고, 정보가 있는 항목만 기준으로 네 적합도 점수를 100점으로 정규화한다.
9. 서버가 `comment_reason`에 실제 입력 근거 ID를 연결한 뒤 개인 적합도와 취약도를 50:50으로 결합하고 다양성 조건을 적용한다.
10. 동일 필터와 다양성 조건으로 규칙 기준선과 AI 혼합 Top 3를 만든다.
11. AI 입력과 분리한 루브릭 판정 정답 라벨로 NDCG@3와 Macro F1을 계산한다.

### 역할 분담

- 시스템 계산: 콘텐츠 유형 선호, 평균 학습 시간 대비 예상 시간, 사용자 수준 대비 콘텐츠 난이도
- AI 판단: 자연어로 된 사용자 목표와 콘텐츠 목표의 의미 관계, 추천 이유 유형, 학생용 추천 문구
- 서버 후처리: 목표 관계의 점수 변환, 적용 가능한 항목만 재환산, 근거 ID 연결, 취약도와 50:50 결합, 최종 순위와 다양성 결정

콘텐츠 유형은 크기 순서가 있는 숫자가 아니라 `quiz`, `video`, `reading`, `practice` 같은 범주값으로 저장한다. 시스템은 이 범주가 선호 유형인지와 해당 유형 완료율을 함께 보고 유형 점수를 `25/22/20/16/8/null` 중 하나로 결정한다. 예상 시간은 사용자 평균 학습 시간과의 비율에 따라 `20/16/10/4/null`, 수준은 같은 과목의 순서형 레벨 차이에 따라 `20/14/5/null`로 계산한다. 프로필이나 비교 정보가 없으면 점수를 추측하지 않고 `null`로 둔다.

최근 학습은 추천 이유 선택에만 쓰며, AI에는 정확한 오답률 대신 `0.5/0.75/1.0`의 구간화된 필요 강도와 수치 없는 설명만 전달한다. 원시 학습 이력, 정확한 취약도·오답률·신뢰도, 평가 정답 라벨은 AI 입력에서 제외한다.

AI가 반환하는 목표 관계는 서버에서 `직접 일치 35점`, `강한 연관 28점`, `간접 연관 18점`, `거의 무관 5점`, `판단 정보 없음 null`로 변환한다. `null` 항목은 임의의 중립값으로 채우지 않고 개인 적합도 분모와 분자에서 함께 제외한다. `evidence_id`는 AI가 만들거나 선택하지 않으며, 서버가 선택된 `comment_reason`과 실제 입력 데이터를 기준으로 연결한다.

05~07의 추천 결과 출처와 08의 API 실행 상태는 서로 다른 정보다. API가 성공하면 05~07은 실시간 AI 응답을 사용하고, API가 없거나 실패하면 `정상 AI 응답 가정 시나리오`를 사용한다. 어느 경우든 08은 실제 API 설정·시도·재시도·성공 또는 실패 상태를 그대로 보여준다.

## 합성 데이터

- 단원 4개, 개념 8개
- 문항-개념 연결 24개
- 추천 콘텐츠 40개
- 사용자 fixture 10명
- 문제 풀이 이벤트 150~200건
- 콘텐츠 학습 이력 60~100건
- 사용자-후보 평가 라벨 93건 (`codex-rubric-judged.v2`)

기획 문서에는 구체적인 10명 fixture 행이 없으므로, 문서에 명시된 신규/기존 전환 단계, 행동 프로필 유무, 짧은 퀴즈·긴 강의 선호, 최근 악화, 희소 이력, 후보 부족, 다양성 시나리오를 모두 포함하도록 합성했다.

## 검증

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm benchmark:ai-coach
pnpm build
```

자동 테스트와 benchmark는 실제 OpenAI API를 호출하지 않는다. 실제 API smoke test는 별도로 요청받았을 때만 수행한다.

benchmark 출력에는 dataset, label, logic, prompt, model 버전과 전체 Macro F1, 규칙/혼합 평균 NDCG@3를 기록한다. AI 혼합 결과가 기준선보다 낮더라도 그대로 출력한다.

## 실제 API 연결 후 확인할 항목

- 사용 모델이 Structured Outputs를 지원하는지
- 후보 ID·개수·순서가 입력과 정확히 일치하는지
- AI가 허용된 `goal_relation`, `comment_reason`, `comment`만 후보별로 반환하는지
- 시스템의 유형·시간·수준 점수와 서버의 목표 관계 점수 매핑, `null` 제외 재환산이 유효한지
- 추천 문구가 초등학생에게 적합한 해요체·80자 이내·최대 두 문장이며, 서버가 연결한 evidence ID와 실제 근거가 일치하고 후보 간 시작·구조·끝맺음이 반복되지 않는지
- AI 호출 실패 시에도 05 AI 혼합·06 상세 추천·07 품질 평가가 동일한 `정상 AI 응답 가정 시나리오`로 이어지고, 실제 실패 상태는 08에 기록되는지
- 데모의 시도당 20초 제한과 1회 재시도가 기록되며, 운영 기획의 2초 목표와 혼동되지 않는지
- 응답 시간과 입력·출력 토큰이 화면에 표시되는지
